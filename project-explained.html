Whispering MVP: přesné vysvětlení, jak tenhle Rust projekt funguje

Tohle je malá desktopová aplikace v Rustu. Po kliknutí na tlačítko začne nahrávat z výchozího mikrofonu, průběžně ukládá audio do dočasného WAV souboru, po zastavení ho připraví do formátu vhodného pro Whisper a zavolá lokální model, který vrátí český přepis. Výsledek se zobrazí v GUI a zároveň se vypíše do standardního výstupu.

GUI: eframe / egui Vstup z mikrofonu: cpal WAV čtení: hound Převzorkování: rubato Přepis: transcribe-rs + Whisper

1. Vstupní bod programu

const MODEL_PATH: &str = ".../ggml-large-v3-turbo.bin";
const LANGUAGE: &str = "cs";

fn main() -> eframe::Result<()> {
    env_logger::init();

    let options = eframe::NativeOptions {
        viewport: egui::ViewportBuilder::default()
            .with_title("Whispering MVP")
            .with_inner_size([720.0, 420.0]),
        ..Default::default()
    };

    eframe::run_native(
        "Whispering MVP",
        options,
        Box::new(|_cc| Ok(Box::new(WhisperingMvpApp::default()))),
    )
}

Co se děje při startu

Skutečný vstupní bod aplikace je src/main.rs. Cargo standardně spouští právě tenhle soubor. Kód nastaví logger, vytvoří nativní okno a předá řízení frameworku eframe, který pak pravidelně volá update smyčku aplikace.

Konstanty MODEL_PATH a LANGUAGE určují, jaký Whisper model se použije a že přepis má běžet v češtině.

Co je jiné oproti TypeScriptu

Návratový typ Result<()> je v Rustu běžný. Místo výjimek se chyba vrací explicitně v typu. To je podobné jako práce s Result v Kotlinu, jen v Rustu je to základní idiom skoro všude.

2. Vlastní chybový typ a zapisování WAV

#[derive(Debug, Error)]
enum AppError {
    #[error("{0}")]
    Message(String),
    #[error("IO error: {0}")]
    Io(#[from] std::io::Error),
    #[error("WAV error: {0}")]
    Wav(#[from] hound::Error),
}

type Result<T> = std::result::Result<T, AppError>;

struct ProgressiveWavWriter {
    writer: BufWriter<File>,
    sample_rate: u32,
    channels: u16,
    bytes_per_sample: u16,
    data_chunk_size_pos: u64,
    riff_chunk_size_pos: u64,
    samples_written: u64,
    last_header_update: Instant,
}

Jak projekt reprezentuje chyby a audio výstup

AppError je vlastní enum pro chyby aplikace. Místo jedné obecné exception má projekt přesně popsané varianty chyb. Díky atributu #[from] se některé typy chyb automaticky převádějí do AppError.

ProgressiveWavWriter ručně zapisuje WAV soubor. To je důležité, protože při startu nahrávání ještě není známá finální délka audia. Writer si proto uloží pozice v hlavičce souboru a průběžně je aktualizuje.

Co je jiné oproti C

Tohle je stále nízkoúrovňová práce se souborem a bajty, ale bez ruční správy paměti. Vlastnictví souboru i writeru hlídá kompilátor, takže nevznikají tak snadno use-after-free problémy.

3. Spuštění nahrávání a worker vlákno

fn start_new_recording(&mut self) -> Result<()> {
    self.close_session()?;

    let file_path = std::env::temp_dir().join(format!("whispering-mvp-{}.wav", ...));
    let host = cpal::default_host();
    let device = host.default_input_device()
        .ok_or_else(|| AppError::Message("No default input device available.".into()))?;

    let config = get_optimal_config(&device, Some(16000))?;
    let writer = Arc::new(Mutex::new(ProgressiveWavWriter::new(
        &file_path, sample_rate, channels,
    )?));

    let (cmd_tx, cmd_rx) = mpsc::channel();
    let worker = thread::spawn(move || {
        let stream = build_input_stream(..., is_recording.clone(), writer_clone)?;
        stream.play()?;

        while let Ok(cmd) = cmd_rx.recv() {
            match cmd {
                RecorderCmd::Start(reply_tx) => { ... }
                RecorderCmd::Stop(reply_tx) => { ... }
                RecorderCmd::Shutdown => break,
            }
        }
    });
}

Jak funguje recording pipeline

Při startu se nejdřív zavře předchozí session, pak se vytvoří dočasný WAV soubor a najde se výchozí mikrofon. Aplikace zvolí vhodnou konfiguraci vstupu a spustí samostatné worker vlákno, které drží audio stream otevřený.

GUI vlákno reaguje na kliknutí uživatele a posílá příkazy.
Kanál mpsc přenáší příkazy start / stop / shutdown.
Worker vlákno obsluhuje stream a zapisuje audio do souboru.

Co je tu rustovské

  • Arc<Mutex<T>> znamená sdílená vlastněná data s bezpečným zamykáním.
  • mpsc::channel() je standardní kanál pro komunikaci mezi vlákny.
  • move || přesune vlastnictví dovnitř closure, takže nehrozí visící reference.

4. Výběr nejlepší konfigurace mikrofonu

fn get_optimal_config(
    device: &Device,
    preferred_sample_rate: Option<u32>,
) -> Result<cpal::SupportedStreamConfig> {
    let target_sample_rate = preferred_sample_rate.unwrap_or(16000);
    let configs: Vec<_> = device.supported_input_configs()?.collect();

    let compatible_configs: Vec<_> = configs
        .iter()
        .filter(|config| supported_formats.contains(&config.sample_format()))
        .collect();

    // 1) preferuj mono + přesně 16 kHz
    // 2) pak libovolný počet kanálů + 16 kHz
    // 3) nakonec nejbližší použitelnou variantu
}

Proč se řeší sample rate a kanály

Whisper chce ideálně mono audio na 16 000 Hz. Proto aplikace nejdřív zkouší najít konfiguraci, která tomu co nejvíc odpovídá už na vstupu. Tím se zmenší množství práce při následném převodu.

Pokud mikrofon přesně takovou konfiguraci neumí, kód vybere nejbližší použitelnou variantu. To je praktická obrana proti různému hardwaru na různých počítačích.

Co je jiné oproti Kotlinu

Typ Option<u32> je explicitní „možná tam hodnota je, možná ne“. Nejde o nullable typ; kompilátor tě nutí obě varianty vyřešit.

5. Audio callback a normalizace vzorků

match sample_format {
    SampleFormat::F32 => device.build_input_stream(
        config,
        move |data: &[f32], _| {
            if is_recording.load(Ordering::Relaxed)
                && let Ok(mut writer) = writer.lock()
            {
                let _ = writer.write_samples_f32(data);
            }
        },
        err_fn,
        None,
    ),
    SampleFormat::I16 => { ... }
    SampleFormat::U16 => { ... }
    _ => Err(...)
}

Co se děje při příjmu audia

Knihovna cpal volá callback pokaždé, když dorazí další blok vzorků z mikrofonu. Podle typu vstupu se vytvoří odpovídající verze callbacku pro f32, i16 nebo u16.

Pokud je nahrávání aktivní, callback zamkne writer a zapíše data do WAV souboru. U integer formátů se hodnoty nejdřív převedou na normalizované f32, aby měl další processing jednotný vstup.

Důležitý rozdíl proti C

Sdílený mutable stav tu nejde používat „jen tak“. Rust tě nutí použít bezpečný mechanismus jako Mutex nebo atomické typy, takže datové závody nejsou jen otázka disciplíny.

6. Správa Whisper modelu

#[derive(Clone)]
struct ModelManager {
    engine: Arc<Mutex<Option<Engine>>>,
    current_model_path: Arc<Mutex<Option<PathBuf>>>,
    last_activity: Arc<Mutex<SystemTime>>,
    idle_timeout: Duration,
}

fn get_or_load_whisper(&self, model_path: PathBuf)
    -> Result<Arc<Mutex<Option<Engine>>>> {
    let needs_load = match (&*engine_guard, &*current_path_guard) {
        (None, _) => true,
        (Some(_), Some(path)) if path != &model_path => true,
        _ => false,
    };

    if needs_load {
        let mut engine = WhisperEngine::new();
        engine.load_model(&model_path)?;
        *engine_guard = Some(Engine::Whisper(engine));
    }
}

Lazy load modelu a idle unload

Whisper model se nenačítá při spuštění aplikace, ale až ve chvíli, kdy je skutečně potřeba přepisovat. To zrychluje start programu a šetří paměť.

ModelManager si pamatuje, jaký model je právě nahraný, a čas poslední aktivity. Pokud je aplikace pět minut nečinná, model se uvolní z paměti.

Rust detail

Option<Engine> nahrazuje nullable objekt. Díky pattern matchingu se dobře modelují všechny možné stavy bez nejasného „může být null“.

7. Přepis WAV souboru na text

fn transcribe_wav_file(model_manager: &ModelManager, file_path: &PathBuf)
    -> Result<String> {
    let audio_data = std::fs::read(file_path)?;
    let samples = extract_whisper_samples_from_wav(audio_data)?;
    if samples.is_empty() {
        return Ok(String::new());
    }

    let engine_arc = model_manager.get_or_load_whisper(PathBuf::from(MODEL_PATH))?;
    let mut params = WhisperInferenceParams::default();
    params.language = Some(LANGUAGE.to_string());
    params.print_progress = false;
    params.suppress_blank = true;

    let mut engine_guard = engine_arc.lock()?;
    let Engine::Whisper(whisper_engine) = engine;
    let result = whisper_engine.transcribe_samples(samples, Some(params))?;
    Ok(result.text.trim().to_string())
}

Samotná transkripce

Po zastavení nahrávání se dočasný WAV soubor načte do paměti, z něj se vytáhnou vzorky, případně se upraví na požadovaný formát a pak se zavolá Whisper. Pokud je audio prázdné, funkce vrátí prázdný řetězec.

Inference běží v samostatném vlákně, takže GUI nezamrzne. Hotový výsledek se pošle zpět přes kanál jako WorkerEvent.

Rust vs Kotlin sealed classes

Řádek let Engine::Whisper(whisper_engine) = engine; je destrukturalizace enumu. Je to podobná myšlenka jako práce se sealed class hierarchií, ale pattern matching je v Rustu přímá součást jazyka.

8. Downmix a převzorkování na 16 kHz

let mono_samples = if channels <= 1 {
    samples_f32
} else {
    samples_f32
        .chunks_exact(channels)
        .map(|chunk| chunk.iter().sum::<f32>() / channels as f32)
        .collect()
};

if spec.sample_rate == 16000 {
    return Ok(mono_samples);
}

let resample_ratio = 16000.0 / spec.sample_rate as f64;
let mut resampler = SincFixedIn::<f32>::new(resample_ratio, 8.0, params, chunk_size, 1)?;

while input_pos < mono_samples.len() {
    let mut chunk = mono_samples[input_pos..end_pos].to_vec();
    if chunk.len() < chunk_size {
        chunk.resize(chunk_size, 0.0);
    }
    let waves_out = resampler.process(&vec![chunk], None)?;
    output_samples.extend_from_slice(&waves_out[0]);
}

Příprava audia pro Whisper

Whisper očekává mono float samples na 16 kHz. Proto kód nejdřív udělá downmix více kanálů na jeden a pak, pokud je potřeba, provede převzorkování pomocí knihovny rubato.

Zpracování po blocích je praktické kvůli resampleru i kvůli kontrolované práci s pamětí. Poslední blok se doplní nulami a výsledný vektor se pak zkrátí na očekávanou délku.

Co je podobné Cčku

Tady už je kód hodně datově orientovaný: vektory, převody typů, výpočty délek, práce po blocích. Rozdíl je v tom, že Rust hlídá indexování, vlastnictví i mutace výrazně přísněji.

9. Update smyčka GUI a uživatelský tok

impl eframe::App for WhisperingMvpApp {
    fn update(&mut self, ctx: &egui::Context, _frame: &mut eframe::Frame) {
        self.model_manager.unload_if_idle();

        while let Ok(event) = self.worker_rx.try_recv() {
            match event {
                WorkerEvent::TranscriptReady(result) => {
                    self.is_transcribing = false;
                    match result {
                        Ok(transcript) => {
                            println!("{transcript}");
                            self.transcript = transcript;
                            self.status = "Done...".into();
                        }
                        Err(err) => self.status = format!("Transcription failed: {err}"),
                    }
                }
            }
        }

        if ui.add_enabled(enabled, egui::Button::new(button_text)).clicked() {
            self.toggle_recording();
        }
    }
}

Jak to celé teče z pohledu uživatele

GUI běží v pravidelné update smyčce. V každém kroku zpracuje nové události z worker vlákna, případně aktualizuje stav aplikace a znovu vykreslí tlačítko, status i textové pole s přepisem.

Uživatelský tok je jednoduchý: Start recording → nahrávání do WAV → Stop recording → background thread spustí přepis → výsledek se vrátí do GUI.

Rust detail

Stav aplikace je přímo ve structu WhisperingMvpApp a metoda update(&mut self, ...) ho mění explicitně. Není v tom skoro žádná magie: jen data a funkce nad nimi.

Celkový mentální model projektu

Architektura jednou větou

GUI vlákno řídí uživatele, worker vlákno drží audio stream, dočasný WAV odděluje nahrávání od přepisu a ModelManager spravuje životní cyklus Whisper modelu.

Nejdůležitější rustovské pojmy

  • Result a Option místo výjimek a nullů.
  • &T a &mut T pro bezpečné reference.
  • Arc + Mutex pro sdílení mezi vlákny.
  • enum + pattern matching pro modelování stavů.

Kde je hlavní logika

Nejvíc důležitých rozhodnutí je ve funkcích start_new_recording, get_optimal_config, extract_whisper_samples_from_wav a transcribe_wav_file.

Co si z toho odnést

Tenhle projekt není obecný framework, ale soustředěná pipeline: nahrát → uložit → převést → přepsat → zobrazit. Rust tu pomáhá hlavně přesným typováním, bezpečnou prací mezi vlákny a explicitním zacházením s chybami.
